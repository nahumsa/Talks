Chemistry with Qubits¶

1) Creating the Qubit Hamiltonian¶

The qubit hamiltonian is of the form:

$$ \mathcal{H} = \sum_{p,q=1}^N t_{pq} c^\dagger_p c_q + \sum_{p,q,r,s=1}^N u_{pqrs} c^\dagger_p c^\dagger_q c_r c_s $$

Thus we need to calculate $t_{pq}$, $u_{pqrs}$ and the fermionic operators.

1.1) Calculating the integrals¶

In order to create the qubit hamiltonian, we need to calculate the one- and two-body integrals this is done in using PySCFDriver. For creating the Qubit Hamiltonian we need three quantities:

  • Basis
  • Molecule
  • Coordinates

Another important quantity that we need to get is the nuclear repulsion energy that will be subtracted from the energy that we find using VQE.

In [2]:
import numpy as np
import matplotlib.pyplot as plt
from qiskit.chemistry.drivers import PySCFDriver, UnitsType

molecular_coordinates = 'Li 0 0 0; H 1.6 0 0'
distance_unit = UnitsType.ANGSTROM
basis = 'sto3g'

driver = PySCFDriver(molecular_coordinates, unit=distance_unit)

molecule = driver.run()

h1 = molecule.one_body_integrals
h2 = molecule.two_body_integrals
nuclear_repulsion_energy = molecule.nuclear_repulsion_energy

1.2) Creating the fermionic operator¶

To create the fermionic operator we use the one- and two-body integrals that we calculated before and use the FermionicOperator class from qiskit.

In [3]:
from qiskit.chemistry import FermionicOperator

ferOp = FermionicOperator(h1=h1, h2=h2)

num_spin_orbitals = molecule.num_orbitals * 2
num_particles = molecule.num_alpha + molecule.num_beta

freeze_list = [0]
remove_list = [-3, -2]

# Convert all negative idx to positive
remove_list = [x % molecule.num_orbitals for x in remove_list]
freeze_list = [x % molecule.num_orbitals for x in freeze_list]
remove_list = [x - len(freeze_list) for x in remove_list]

# Update the idx in remove_list of the idx after frozen, 
# since the idx of orbitals are changed after freezing

remove_list += [x + molecule.num_orbitals - len(freeze_list)  for x in remove_list]
freeze_list += [x + molecule.num_orbitals for x in freeze_list]

We can reduce the number of qubits, freezing some of the modes which will always be full.

In [4]:
ferOp_f, energy_shift = ferOp.fermion_mode_freezing(freeze_list)

num_spin_orbitals -= len(freeze_list)
num_particles -= len(freeze_list)

We can also remove some of the orbitals which the energy would be too high for the ground state, and thus would not be accessed.

In [5]:
ferOp_fr = ferOp_f.fermion_mode_elimination(remove_list)
num_spin_orbitals -= len(remove_list)

Solving the operator exactly we have the smallest eigenvalue:

In [17]:
import warnings
warnings.filterwarnings("ignore")

from qiskit.aqua.algorithms import ExactEigensolver

def exact_solver(qubitOp):
    ee = ExactEigensolver(qubitOp)
    result = ee.run()
    ref = result['energy']
    # print('Reference value: {}'.format(ref))
    return ref

ref = exact_solver(qubitOp_t)
print(f'Exact energy is: {np.round(ref,4)}')

2) Applying the VQE for finding the ground state¶

2.1) Running on a ideal device¶

Running on an ideal device we do not need to worry about decoherence and the coupling mapping, thus I shall use the best possible ansatz which is the UCCSD, this will show us that this ansatz is impossible to work on current devices.

For the UCCSD we usually choose the initial state to be the Hartree-Fock solution which can be calculated efficiently on a classical computer.

In [18]:
from qiskit.aqua import QuantumInstance
from qiskit.aqua.algorithms import VQE
In [19]:
from qiskit import BasicAer
backend = BasicAer.get_backend("statevector_simulator")
In [20]:
from qiskit.chemistry.components.variational_forms import UCCSD
from qiskit.chemistry.components.initial_states import HartreeFock

initial_state = HartreeFock(num_spin_orbitals,
        num_particles,
        qubit_mapping='parity') 

var_form = UCCSD(
        num_orbitals=num_spin_orbitals,
        num_particles=num_particles,
        initial_state=initial_state,
        qubit_mapping='parity')

2.2) Running on a Noisy Device¶

Now that we are running on a noisy device with limited connection, we should use a more efficient ansatz

Let's define our noise model based on the ibmq_valencia chip which is a quantum computer with QV16 and 5 qubits.
